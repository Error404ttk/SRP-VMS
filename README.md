# 🚑 SRP-VMS (SPH Vehicle Management System)
> **ระบบบันทึกการใช้รถราชการ โรงพยาบาลสารภี (v2.0.0-security-dynamic)**  
> พัฒนาขึ้นด้วยสถาปัตยกรรมแบบผสมผสาน (Hybrid Architecture) รองรับการรันแบบ On-Premises/Cloud ด้วย Node.js Express และรองรับสแตนด์อะโลนบน Google Apps Script (GAS) Web App ⚡

---

## 📂 โครงสร้างของโปรเจกต์ (Project Directory Structure)

```text
├── / (Root Directory)               # ซอร์สโค้ดฝั่ง Google Apps Script (.gs และ .html)
├── /sph-server                      # โครงสร้างระบบหลักหลังบ้าน (Node.js Express Server)
│   ├── /controllers                 # เลเยอร์รับค่าความปลอดภัยและประมวลผลคำขอ
│   ├── /public                      # ไฟล์ทรัพยากรหน้าบ้าน (Assets)
│   ├── /routes                      # ระบบจัดเส้นทางนำทาง (Web Pages & API RPC)
│   ├── /services                    # เลเยอร์เชื่อมต่อนอกระบบ เช่น Google Sheets & Caching
│   ├── /utils                       # ยูทิลิตีระบบ เช่น ตรวจสอบสิทธิ์, บันทึกรุ่น, และแฮชรหัสผ่าน
│   ├── /views                       # เทมเพลตฝั่งเซิร์ฟเวอร์แบบ EJS Layout
│   └── server.js                    # จุดเริ่มสตาร์ทการรันเซิร์ฟเวอร์หลัก Node.js
├── /provider-api-proxy              # ระบบ Proxy เชื่อมโยงบัญชีและอำนาจสิทธิ์การเข้าใช้งาน
└── README.md                        # คู่มือการติดตั้งระบบฉบับนี้
```

## ⚙️ ขั้นตอนการติดตั้งและเริ่มการทำงานระบบ (Getting Started & Setup Guide)

เมื่อคุณทำการ Clone โปรเจกต์นี้ลงไปในเครื่องคอมพิวเตอร์ของคุณแล้ว ให้ทำตามขั้นตอนการเซ็ตอัปตามระบบที่คุณเลือกใช้งานดังต่อไปนี้ครับ:

---

### 🖥️ วิธีการติดตั้งที่ 1: ระบบหลังบ้านหลัก Node.js Express (`/sph-server`)

ระบบหลักหลังบ้านของ Node.js จะติดต่อฐานข้อมูลผ่าน Google Sheets API โดยมีขั้นตอนการเซ็ตอัปดังนี้:

#### 1. ติดตั้งแพ็กเกจไลบรารีระบบ:
เปิดเทอมินัลนำทางเข้าไปในโฟลเดอร์ `sph-server` และติดตั้งโมดูล dependencies ทั้งหมด:
```bash
cd sph-server
npm install
```

#### 2. ตั้งค่าไฟล์สภาพแวดล้อมความปลอดภัย (Environment Variables):
ก๊อปปี้ไฟล์ต้นแบบคอนฟิกไปเป็นไฟล์ใช้งานจริง:
```bash
cp .env.example .env
```
เปิดไฟล์ `.env` แล้วป้อนข้อมูลระบุความลับต่าง ๆ ให้ครบถ้วน:
```ini
PORT=9998
JWT_SECRET=ใส่คีย์ลับสำหรับเข้ารหัสโทเคนขั้นต่ำ_32_ตัวอักษร
SPREADSHEET_ID=ใส่รหัส_ID_ของ_Google_Spreadsheet_ที่เป็นฐานข้อมูล

# ข้อมูลเชื่อมต่อกับ Google Service Account API
GOOGLE_SERVICE_ACCOUNT_EMAIL=อีเมลบริการของบอท_Service_Account
GOOGLE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\nคีย์ลับส่วนตัวแถวเดียว\n-----END PRIVATE KEY-----"

# ข้อมูลการตั้งค่าระบบและ SSO Provider ID (หากเปิดใช้งาน)
PROVIDER_CLIENT_ID=รหัสแอปจากทางกระทรวงสาธารณสุข
PROVIDER_CLIENT_SECRET=คีย์ลับจากทางกระทรวงสาธารณสุข
PROVIDER_REDIRECT_URI=http://localhost:9998/login
PROVIDER_SSO_URL=https://moph-id-uat.moph.go.th/oauth2/authorize
```

#### 3. สตาร์ทเริ่มรันเซิร์ฟเวอร์ในสภาพแวดล้อมจำลองพนักงาน (Development):
```bash
npm run dev
# หรือรันผ่านคำสั่งตรง: node server.js
```
เปิดบราวเซอร์เข้าสู่หน้าเว็บได้ทันทีที่ URL: **`http://localhost:9998`** 🎉

### ☁️ วิธีการติดตั้งที่ 2: ระบบ Google Apps Script (Standalone Web App)

สำหรับต้องการรันโปรเจกต์สแตนด์อะโลนบนระบบคลาวด์ของ Google Drive โดยติดต่อฐานข้อมูลในสเปรดชีตของบัญชีตนเอง:

#### 1. การส่งซอร์สโค้ดขึ้น Apps Script:
* **ทางเลือกที่ 1 (สะดวกและปลอดภัยสูง):** นำซอร์สโค้ดไฟล์ `.gs` และ `.html` ในระดับโฟลเดอร์นอกสุด (Root) คัดลอกไปสร้างและวางทับในหน้าจอโปรแกรมสร้างสคริปต์ของ Google Apps Script
* **ทางเลือกที่ 2 (ผ่าน Clasp CLI):** สั่งล็อกอินและทำการส่งไฟล์ขึ้นผ่านคำสั่งคลาสป:
  ```bash
  clasp login
  clasp push
  ```

#### 2. การตั้งค่าสเปรดชีตฐานข้อมูลในตัวแปรระบบ (Script Properties):
หลังจากนำโค้ดขึ้นแล้ว ให้เข้าไปที่หน้าจอแก้ไขสคริปต์ของ Google Apps Script:
1. ไปที่ **การตั้งค่าโครงการ (Project Settings)** (สัญลักษณ์รูปฟันเฟืองด้านซ้าย)
2. เลื่อนลงไปหัวข้อ **คุณสมบัติของสคริปต์ (Script Properties)**
3. กดคลิกปุ่ม **เพิ่มคุณสมบัติของสคริปต์ (Add script property)** โดยกำหนดตัวแปรดังนี้:
   * **ชื่อคุณสมบัติ (Property Name):** `SPREADSHEET_ID`
   * **ค่าของคุณสมบัติ (Value):** `ใส่รหัส ID สเปรดชีตของคุณที่ต้องการใช้เป็นฐานข้อมูล`
4. กดบันทึกการแก้ไข

#### 3. การใช้งานครั้งแรกและการปล่อยตัวแอปพลิเคชัน (Deployment):
1. ในหน้าแก้ไขสคริปต์ ให้เลือกฟังก์ชัน `ensureDatabaseReady` ในไฟล์ `Code.gs` แล้วกดปุ่ม **รัน (Run)** เพื่อให้ระบบสร้างชีต ตารางข้อมูลพนักงานเริ่มต้น (Seed) และตารางพาหนะให้อัตโนมัติในสเปรดชีตของโรงพยาบาลคุณ
2. ทำการกดปุ่ม **การทำให้ใช้งานได้ (Deploy)** -> **การทำให้ใช้งานได้ใหม่ (New deployment)**
3. เลือกประเภทเป็น **เว็บแอป (Web app)**
4. กำหนดสิทธิ์ให้สิทธิ์การทำงานเป็นบัญชีของคุณ และกำหนดให้ทุกคนเป็นพนักงานสิทธิ์เข้าถึง (Who has access: **Anyone**)
5. กดปุ่มบันทึกและคัดลอก URL ของ Web App ไปใช้งานได้ทันทีครับ! 🚀

## 🔒 นโยบายความปลอดภัยข้อมูลส่วนบุคคลและข้อมูลวิจัย (PDPA & Data Security Rules)

* ระบบได้รับการปรับปรุงและทบทวนความปลอดภัยล่าสุด (**v2.0.0-security-dynamic**) ซึ่งผ่านเกณฑ์ตรวจสอบโค้ด **Code & Security Audit เรียบร้อย**
* มีระบบ **Data Masking** ปิดบังข้อมูลอ่อนไหวระดับวิกฤต เช่น รหัสบัตรประชาชนของผู้ใช้งาน รวมถึง HCODE จะถูกปิดการแสดงผลอัตโนมัติหากผู้ใช้งานไม่ใช่ระดับผู้ดูแลระบบ (Admin)
* ข้อมูลรหัสพนักงานถูกจัดแฮชเชิงลึก 12,000 รอบด้วยกลไก **SHA-256 Iterative Salt Hashing (`sha256i`)** แข็งแกร่งต่อนโยบายความปลอดภัยไซเบอร์สูงสุด
